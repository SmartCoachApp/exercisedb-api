# Internationalization (i18n)

Multi-language support for the exercises API. Clients request data in different languages via `?lang=es`.

## User Capabilities

- Consumers can request any endpoint with `?lang=es` to receive results in Spanish
- English (`en`) is the default when no `lang` param is provided
- Catalog endpoints return translated body parts, muscles, and equipment names
- Exercise responses return translated name, description, and instructions
- Path params (bodyPart, target, equipment, id) always use English values regardless of `lang`

## API Contract

The `lang` param applies to all endpoints:

```
GET /api/v1/exercises?lang=es
GET /api/v1/exercises/bodyPart/waist?lang=es
GET /api/v1/exercises/target/abs?lang=es
GET /api/v1/exercises/name/sentadilla?lang=es
GET /api/v1/exercises/bodyPartList?lang=es
GET /api/v1/exercises/targetList?lang=es
GET /api/v1/exercises/equipmentList?lang=es
```

Supported values: `en` (default), `es`. Invalid values fall back to `en` silently.

## Translation Data

### Catalog translations
Static dictionaries mapping English → Spanish:
- 10 body parts (e.g. `"waist"` → `"cintura"`)
- 19 target muscles (e.g. `"abs"` → `"abdominales"`)
- 28 equipment types (e.g. `"dumbbell"` → `"mancuerna"`)

### Exercise translations (1,324 exercises)
Per-exercise translations keyed by `id`:
- Exercise name
- Description
- Instructions (array of steps)

## Data Architecture

```
src/data/
├── exercises.json              # English source of truth
├── body_parts.json
├── targets.json
├── equipment.json
└── i18n/
    └── es/
        ├── catalogs.json       # body parts, targets, equipment translations
        ├── exercises_001.json  # exercises 0001–0100 translations
        ├── exercises_002.json  # exercises 0101–0200 translations
        └── ...
```

### catalogs.json format
```json
{
  "bodyParts": { "waist": "cintura", "chest": "pecho" },
  "targets":   { "abs": "abdominales", "biceps": "bíceps" },
  "equipment": { "dumbbell": "mancuerna", "barbell": "barra" }
}
```

### exercises_NNN.json format
```json
{
  "0001": {
    "name": "Sentadilla 3/4",
    "description": "El sit-up 3/4 es un ejercicio abdominal...",
    "instructions": ["Acuéstate boca arriba...", "Contrayendo el abdomen..."]
  }
}
```

## Constraints

- English is always the internal key and source of truth
- URL path params remain English-only
- Translation files are loaded and merged into a single in-memory map at startup
- `images` signed URLs are language-independent
- `id`, `difficulty`, `category` are never translated (universal values)
